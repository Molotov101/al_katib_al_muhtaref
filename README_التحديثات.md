# الكاتب المحترف — النسخة النهائية المصححة

## ما تغيّر في هذه النسخة تحديداً (مقارنة بما اقترحته الرسالة السابقة)

كنت تطلب تنفيذ حلّين: تحميل خطوط عبر `curl` من GitHub، وربط ترجمة عبر
LibreTranslate بدون مفتاح. تحققتُ فعلياً من الاثنين قبل التنفيذ ووجدت
مشكلتين حقيقيتين، فاستبدلتهما بحل أقوى وأضمن:

### 1. الخطوط — لم تعد تحتاج تحميل أي ملف يدوياً
- **المشكلة:** رابط `raw.githubusercontent.com/.../Cairo-Regular.ttf` أصبح
  معطلاً (404) لأن مستودع `google/fonts` حوّل خط Cairo إلى Variable Font
  واحد بلا ملفات Regular/Bold منفصلة. كما أن بيئة التنفيذ هنا لا تملك
  اتصالاً بالإنترنت في الطرفية أصلاً (`host_not_allowed`).
- **الحل المطبَّق:** استُبدل نظام الخطوط بالكامل بحزمة `google_fonts`
  (كانت موجودة أصلاً في `pubspec.yaml`). الخط يُجلب تلقائياً وقت تشغيل
  التطبيق على جهاز المستخدم من خوادم Google، ويُخزَّن محلياً بعدها.
  **لا حاجة لأي تحميل يدوي أو تسمية ملفات بدقة.**
- ملف جديد: `lib/core/utils/font_helper.dart`
- **يتطلب اتصالاً بالإنترنت أول مرة يُعرض فيها كل خط فقط.**

### 2. الترجمة — أصبحت فعلية عبر مزوّد مجاني يعمل بالفعل
- **المشكلة:** تحققت من توثيق LibreTranslate الرسمي — الخدمة العامة على
  `libretranslate.com` أصبحت تتطلب الآن مفتاح API **مدفوعاً** بعد تحديث
  حديث من مالكي الخدمة. الكود المقترح سابقاً كان سيفشل بخطأ 400.
- **الحل المطبَّق:** استُبدلت بخدمة **MyMemory Translation API**
  (`api.mymemory.translated.net`) — مجانية فعلياً وبلا أي مفتاح حتى الآن،
  مع تقسيم تلقائي للنصوص الطويلة (لأن حدها ~500 حرف للطلب الواحد).
- ملف محدَّث بالكامل: `lib/services/translation_service.dart`
- شاشة الترجمة (`translation_screen.dart`) أصبحت تستدعيها فعلياً بدل
  المحاكاة الوهمية، مع معالجة أخطاء حقيقية (رسالة واضحة عند فشل الاتصال).
- **يتطلب اتصالاً بالإنترنت وقت الترجمة فقط.**

## خطوات التشغيل (بدون تغيير)

```bash
flutter pub get
flutter pub run build_runner build --delete-conflicting-outputs
flutter run
```

لا حاجة لأي خطوة تحميل يدوية إضافية بعد الآن — كل شيء يعمل من الصندوق
بمجرد توفر اتصال إنترنت على جهاز المستخدم وقت الاستخدام الفعلي (وليس
وقت البناء).
